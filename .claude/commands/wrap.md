이번 세션에서 실제로 한 작업을 기준으로 프로젝트 문서를 갱신한다. **없던 일을 지어내지 말고**, 이 세션의 대화·변경·검증 결과에 근거해서만 작성한다 (근거 없는 항목 금지 — CLAUDE.md §1).

## 참고: 현재 저장소 상태 (자동 주입)
- 세션 주제(사용자 입력): $ARGUMENTS
- 변경된 파일:
!`git status --short 2>/dev/null || echo "(git 저장소 아님 — 대화 내용 기준으로 작성)"`
- 최근 커밋:
!`git log --oneline -15 2>/dev/null || echo "(로그 없음)"`
- 변경 규모:
!`git diff --stat 2>/dev/null | tail -30 || true`

## 갱신 절차
1. **기존 문서를 먼저 읽는다**: `docs/SUMMARY.md`, `docs/TODO.md`, `docs/details/` 의 파일들. (없으면 템플릿 형태로 새로 만든다.)
2. **`docs/details/<오늘날짜>-<주제>.md` 를 추가/갱신한다.**
   - 파일명 날짜는 오늘 날짜(YYYY-MM-DD) 사용.
   - 구성: 배경/목표 → 결정·구현 내용(`파일:줄` 인용) → 검증 결과(실제 실행/출력) → 남은 작업.
   - 같은 주제의 파일이 이미 있으면 덮어쓰지 말고 이어서 갱신한다.
3. **`docs/SUMMARY.md` 를 갱신한다** (누적 요약, 덮어쓰지 말고 병합).
   - "현재 상태"를 이번 세션 결과로 최신화.
   - "최근 변경 이력" 표에 한 줄 추가: `| 오늘날짜 | 세션 요약 | details 파일 링크 |`.
4. **`docs/TODO.md` 를 갱신한다.**
   - 이번 세션에 완료한 항목은 "완료(최근)"로 이동.
   - 새로 생긴 후속 작업·미결·블로커를 해당 섹션에 추가.

## 규칙
- **검증된 것만 "완료"로 적는다.** 실행/확인 안 한 것은 TODO의 "확인 필요"에 남긴다.
- **추측·과장 금지.** 실패했거나 미검증이면 그대로 기록한다.
- **수술적 갱신.** 관련 없는 기존 문서 내용을 임의로 고치거나 삭제하지 않는다. 이번 세션과 연결된 부분만 갱신한다.
- 마지막에 **갱신한 파일 목록**과 각 파일의 핵심 변경 한 줄씩을 요약해 보고한다.
