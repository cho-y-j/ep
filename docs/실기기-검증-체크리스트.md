# 실기기 검증 체크리스트 (통합본)

> 코드·에뮬레이터 수준 검증은 전부 완료된 항목들입니다. 아래는 **실제 하드웨어(갤럭시 폰 + 갤럭시 워치5 + NFC 태그 + 현장)** 없이는 확정 불가한 항목만 모았습니다.
> 준비물: 갤럭시 폰 2대 이상(작업자·동료·BP 역할), 갤럭시 워치5(개인 지급 1대), NFC 태그 스티커, 커프 혈압계, Firebase 실제 `google-services.json`(폰·워치 각 모듈).
> 작성: 2026-07-19 (P5-W0~W4 검수 산출 통합)

## A. 기반 설정 (최초 1회)
- [ ] A1. 실제 `google-services.json`을 `app/phone-app/app/`·`app/watch-app/app/`에 배치 후 릴리즈 빌드
- [ ] A2. 백엔드 `firebase-admin-key.json` 배치 → FCM 실발송 활성 (dev는 현재 로그 전용)
- [ ] A3. 폰 최초 실행: 알림·위치·**BLE(광고/스캔)**·카메라 권한 허용, Android 14는 전체화면 인텐트 권한 확인
- [ ] A4. 워치 페어링: 서버주소(`https://skep.on1.kr`)·출근코드 → WORKER_ID(person id) 하달 확인
- [ ] A5. 카카오 콘솔에 서비스 도메인 등록 → 웹 지도·폰 출근맵 실렌더 확인 (도메인 확정 후)

## B. 안전 알림 3등급 (P3b)
- [ ] B1. 긴급(EMERGENCY): **무음모드+DND 상태**에서 알람음·반복 진동·풀스크린·TTS 한국어 읽기 확인
- [ ] B2. 잠금화면에서 풀스크린 알림 표출 + [확인] 탭 → 서버 ack 반영(관제 확인률 갱신)
- [ ] B3. 주의/안내 등급이 각각 헤드업/일반으로 차등 표출되는지
- [ ] B4. 5분 미확인 시 재알림 1회 + 관리자 에스컬레이션 실수신

## C. 출근·NFC·확인서 (기존 흐름)
- [ ] C1. 출근 체크인: 위치 확인·출근사진 업로드·지오펜스 정확도(현장 경계)
- [ ] C2. NFC 법정점검: 실태그 → 점검표 오픈(태그 없이 열리지 않음 확인) → 서명 저장
- [ ] C3. 퇴근 → 작업확인서 자동생성 → 폰 서명(작업자) → BP 폰 서명 → COMPLETED
- [ ] C4. 워치 착용 상태로 10시간 근무 시뮬레이션: **배터리 소모율 측정** (목표 시간당 ≤5%)

## D. 워치 안전 체인 (P5-W0~W1)
- [ ] D1. 워치 3색 상태: 평상 시 10분 묶음 전송(폰 로그로 주기 확인)·폭염특보 시 5분 주기 전환
- [ ] D2. **워치 낙상/수동호출 → 피재자 본인 폰이 서버와 무관하게 로컬 경보**(진동+풀스크린) — (a)폰 백그라운드/잠금에서도 발동 (b)토큰 만료 상태에서도 발동 (c)[확인]으로 해제 (d)`/skep/status` 상시 전송으로는 오발동 없음
- [ ] D3. 워치 45초 자가취소 카운트다운 실시간 표시·[괜찮아요]로 취소
- [ ] D4. 데드맨: 워치 전원 끄고 30분 대기 → 폰 "워치 확인" 요청 수신 → 미확인 5분 후 관리자 통보
- [ ] D5. 오프바디: 워치 벗은 상태 → 관제에 "미착용" 구분 표시(두절과 다른 라벨)
- [ ] D6. 개인 베이스라인: 캘리브레이션 후 서버 동기화 → `/watch-policy`로 개인 심박 대역 하달 → 워치 온디바이스 임계 갱신 확인
- [ ] D7. 배터리 잔량이 관제 워치 타일에 표시·20% 이하 강조

## E. 긴급 대응 체인 (P5-W2 — 신규)
- [ ] E1. **파인드미**: 긴급 확정 시 피재자 폰 사이렌 최대음량(무음모드에서도)·**플래시 4Hz 점멸**·화면 흑백 점멸 — 어두운 곳에서 시인성 확인
- [ ] E2. [구조자 도착·해제] 버튼 → 사이렌·플래시·BLE 광고 전부 즉시 중지 (준비 직후 눌러도 사이렌 잔류 없음 — 레이스 픽스 확인)
- [ ] E3. 동료 폰: `제가 갑니다` 풀스크린 수신(피재자 이름·거리) → 응답 → 관제에 "N명 응답·이동중" 표시
- [ ] E4. **근접 게이지**: 동료 폰이 피재자 BLE 신호 강도로 게이지 상승·진동 피드백 — 실내/층간에서 동작 확인
- [ ] E5. 60초 무응답 → 통보 범위 확대+관리자 알림 실수신

## F. BLE 대리중계 (P5-W3 — 신규·핵심 리스크)
- [ ] F1. ★**백그라운드 스캔이 스캔응답 serviceData(피재자 신원)를 포착하는지** — 기종별 BLE 칩셋 편차 있음. 실패 시 중계가 조용히 skip됨 → 반드시 2개 이상 기종에서 확인
- [ ] F2. 워치 단독 SOS: 폰을 꺼두고 워치 긴급 발동 → 워치 BLE 광고 → **지나가는 제3자 폰이 수신 → 서버 대리중계** → 관제 경보 생성(추정 위치 포함)
- [ ] F3. 제3자 폰 재부팅 후에도 스캔 자동 재등록(제조사 배터리 정책 확인 — 삼성/샤오미 별도)
- [ ] F4. 중계자 폰에 무음·무진동의 조용한 알림만 표시되는지
- [ ] F5. 워치 BLE peripheral(광고) 지원 기종 확인 — 미지원 기종은 로그만 남고 skip됨

## G. 건강 3겹 (P5-W4 — 신규)
- [ ] G1. 커프 혈압계 측정값 입력(수기/BLE) → 임계 초과 시 관리자 통보·기록
- [ ] G2. 고위험군(HIGH) 태깅 → 워치 전송주기 자동 상향(YELLOW) 확인
- [ ] G3. 삼성 헬스 혈압(워치5 스팟 측정)이 Health Connect로 흘러오는지 — 한국 규제 환경 실확인 (안 되면 수기 입력 폴백)
- [ ] G4. 과로 경고: 연속 야간 3일 데이터 상태에서 본인+관리자 알림 수신

## H. 워치 카운트다운·UI (P4f)
- [ ] H1. 워치 ACK 화면 카운트다운 실시간 틱(45초/5분) — 화면 꺼졌다 켜도 이어지는지(재진입 시 리셋 근사치 확인)
- [ ] H2. 폰 13sp 본문·48dp 뒤로 버튼 실기기 가독·터치감
