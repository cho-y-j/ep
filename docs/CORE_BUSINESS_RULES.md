# SKEP v2 핵심 비즈니스 전제

이 문서는 SKEP v2 설계에서 절대 흔들리면 안 되는 핵심 전제를 별도로 기록한다.

## 1. BP사가 현장을 구성한다

현장에 장비공급사와 인력공급사가 미리 존재하고 BP사가 그중에서 고르는 구조가 아니다.

정확한 구조는 아래와 같다.

```text
BP사
→ 현장 생성
→ 해당 현장에 참여시킬 장비공급사 선정
→ 해당 현장에 참여시킬 인력공급사 선정
→ 선정된 공급사들을 현장 참여 업체로 구성
→ 그 공급사들의 장비/인원을 기반으로 작업계획서 생성
```

따라서 `site_participants` 또는 동등한 개념은 필수다.

## 2. 공급사는 자기 자원을 스스로 등록한다

장비공급사는 자기 장비를 직접 등록한다.

인력공급사는 자기 인원을 직접 등록한다.

BP사가 장비/인원을 직접 소유하거나 임의로 생성하는 것이 기본 흐름이 아니다. BP사는 현장을 만들고, 그 현장에 참여할 공급사를 선정하고, 작업계획서에서 참여 공급사의 자원을 선택한다.

## 3. 작업계획서는 현장 참여 공급사를 기반으로 만들어진다

작업계획서에서 선택 가능한 장비/인원은 전체 장비/전체 인원이 아니다.

반드시 아래 조건을 만족해야 한다.

```text
작업계획서의 현장
→ 그 현장에 참여 중인 장비공급사/인력공급사
→ 해당 공급사가 등록한 장비/인원
```

즉 작업계획서는 BP사가 구성한 현장 참여 업체들의 자원을 기반으로 생성된다.

## 4. 작업계획서 생성 시 서류 상태가 핵심이다

SKEP v2에서 서류 관리는 보조 기능이 아니라 핵심 기능이다.

작업계획서에 장비/인원을 넣을 때 반드시 확인해야 한다.

- 필수 서류가 모두 등록되어 있는가
- 서류 만료일이 유효한가
- D-30, D-7, 만료 상태인가
- 검증이 필요한 서류가 검증 완료되었는가
- 만료 또는 누락 시 배정을 차단해야 하는가
- 강제 진행이 허용되는 경우 사유가 기록되었는가

## 5. 이전 현장 투입 이력은 우선순위에 반영한다

BP사가 장비공급사, 인력공급사, 장비, 인원을 고를 때 과거 해당 현장에 투입된 이력이 중요하다.

해당 현장에 한 번이라도 투입된 장비/인원은 다음 이유로 우선순위가 높다.

- 이미 현장 투입 경험이 있다.
- 현장 기준으로 서류가 준비되었을 가능성이 높다.
- BP가 다시 선택할 가능성이 높다.
- 작업계획서 작성 속도가 빨라진다.

따라서 후보 목록에는 반드시 아래 표시가 필요하다.

```text
이전 투입
신규
서류완비
서류누락
만료임박
만료
검증완료
사용제한
```

## 6. 장비/인원의 현재 현장 표시가 필요하다

장비와 인원은 한 번에 하나의 현장에만 배치된다.

다만 언제든 현장 이동이 가능해야 한다.

따라서 현재 현장을 빠르게 표시하는 값과 이동 이력을 모두 관리해야 한다.

```text
equipment.current_site_id
persons.current_site_id
equipment_site_assignments
person_site_assignments
```

## 7. 가동률 퍼센트보다 투입 캘린더가 중요하다

초기 설계에서 장비 가동률 TOP 5 같은 분석 지표는 핵심이 아니다.

더 중요한 것은 아래 정보다.

- 어느 날
- 어느 현장에
- 어떤 장비가
- 어떤 인원이
- 몇 시부터 몇 시까지
- 어떤 작업계획서에 따라 투입되었는가

따라서 대시보드는 퍼센트 중심의 가동률보다 작업계획서, 투입 일정, 투입 이력, 서류 위험을 우선한다.

## 8. 사용 제한은 필요하지만 강제 진행도 가능해야 한다

서류가 만료되거나 필수 서류가 누락된 장비/인원은 기본적으로 작업계획서 배정을 막아야 한다.

다만 ADMIN은 강제 진행이 가능해야 한다.

BP도 정책에 따라 강제 진행을 허용할 수 있지만, 이 경우 반드시 사유를 기록해야 한다.

```text
정상: 배정 가능
D-30/D-7: 경고 표시
만료/누락: 기본 차단
강제 진행: 사유 기록 후 진행
```

## 9. 대시보드의 핵심은 문제 발견이다

대시보드는 단순 통계 화면이 아니다.

관리자가 한눈에 확인해야 하는 것은 아래다.

- 장비 서류 문제
- 인원 서류 문제
- 만료 임박
- 만료됨
- 미검증
- 오늘/이번 주 작업계획서
- 현장별 참여 업체와 투입 자원
- 최근 알림

## 10. 최종 기준 문장

SKEP v2는 BP사가 현장을 만들고, 그 현장에 참여할 장비공급사와 인력공급사를 선정한 뒤, 공급사들이 등록한 장비와 인원을 서류 적합성 기준으로 검증하여 작업계획서를 생성하는 시스템이다.
