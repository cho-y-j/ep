## 2. 최종: 사용자 중심 전 권한 UX 검토

각 주체가 **실제로 앱을 쓸 때**의 흐름 적합성 / 더 편리한 자동화 / 더 편한 화면.

### BP (발주)
- **지금 흐름**: 대시보드 처리대기 큐(투입요청·서류심사·서명대기)로 시작 → 견적 단계칩으로 진행상황 → 네비 4그룹으로 견적·선정/계획서·서명/투입·현장/서류·검사 구획.
- **흐름 적합성**: 파이프라인 3~6단계(서류 수령→검사→투입 요청)를 관장하는 데 필요한 화면이 이제 그룹으로 정리됨. **여전한 이음새**: 검사 신청(ResourceCheck)↔날짜통보(SafetyInspection) 두 도메인 분리.
- **더 자동화**: 검사 자동추천(자원종류별) + 두 검사도메인 연결 + BP용 현장 자원 준비현황 롤업(readiness의 BP 버전).
- **더 편한 화면**: 대시보드를 관제화면으로(현장별 파이프라인 보드), 4그룹의 "기타"를 정리.

### 장비공급사
- **지금 흐름**: 받은견적→후보배지로 안전한 선택→응찰(OT 포함)→선정 시 배차초안 1클릭 발송→투입대기 위젯/파이프라인 보드로 상태확인→정산·거래내역서 다운로드.
- **흐름 적합성**: 파이프라인 1·2·3·7·9단계가 매끄러워짐(배차초안·OT프리필·readiness·거래내역서·서류허브). **여전한 이음새**: 서류 5채널은 허브로 묶였으나 개별 메뉴도 공존.
- **더 자동화**: 배차+서류묶음 통합발송, 거래내역서 월마감 자동생성·알림.
- **더 편한 화면**: 서류허브를 기본 진입점으로(개별 메뉴 정리), 자원 파이프라인 보드를 자원관리의 홈으로.

### 인력공급사
- **지금 흐름**: 장비공급사와 동일 + **인력 OT 단가**가 이제 정산에 반영(응찰 OT율→거래내역서까지).
- **흐름 적합성**: 인력 OT 공식 완성으로 정산 정확도 상승. readiness에 인력 자격/교육 상태 포함.
- **더 자동화**: 자격·안전교육 만료 임박 인력 자동 리마인드(만료 스케줄러 확장), 근무일수·OT 작업확인서 자동파생(적용됨).
- **더 편한 화면**: 파이프라인 보드에 인력 자격 유효기간 표시(장비의 검사에 대응).

### 협력사 (장비 협력사)
- **지금 흐름**: (부모 대행등록 or 자가가입→부모승인)→로그인→본인 장비·인력·서류 **전체 관리(수정 포함)**→투입대기/파이프라인 확인→본인 정산·거래내역서(본인 격리).

- **더 자동화**: 협력사 온보딩 위저드(가입→승인→링크 서류업로드 순서안내), 부모용 자식 롤업(자식별 readiness·서류만료·가입대기).
- **더 편한 화면**: SubSuppliersPage를 "협력사 관제"로(승인 UI 추가됨). 협력사 본인 화면은 현행 유지가 정답.

### 작업자 (운전자/인력)
