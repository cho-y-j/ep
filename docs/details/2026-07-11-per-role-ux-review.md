# 주체별 UX·자동화·화면구성 검토 (최종)

2026-07-11. 파이프라인 스펙(`deployment-pipeline-spec.md`) + 4주체 기능테스트 + A/B/C/D·safe-8·E/D/A 구현(전부 독립 QA 검증) 기준. 각 주체: ①작업흐름 적합성 ②더 편리한 자동화 ③더 편한 화면 구성.

## 이번 세션에 이미 해소된 것 (출발점)
협력사 대행 전체수정(A)·자가로그인+부모승인(B)·투입대기 가시성(C)·거래내역서(D) + 후보배지(E)·근무일수 자동파생(D')·배차초안(A')·서명대기·OT프리필·단계칩·일괄수락·만료알림·거짓카피 수정. → 아래는 그 위에서의 **다음 레버**.

---

## 1. BP (발주)

**작업흐름 적합성** — BP는 파이프라인의 3~6단계(서류 수령→검사 신청/일정/승인→투입 요청)와 견적·작업계획서·서명·정산을 모두 관장. 지금은 **처리대기 위젯 + 견적 단계칩**으로 "지금 뭘 할지"가 일부 보임. 끊기는 곳: (1) 검사 **신청**(ResourceCheck)과 **날짜 통보**(SafetyInspection)가 **다른 두 도메인**이라 BP가 양쪽을 따로 관리, (2) "투입 대기"가 BP 화면엔 자원 롤업으로 안 보임, (3) 견적·선정·계획서·투입·서류가 병렬 메뉴로 흩어짐.

**더 편리한 자동화**
- 검사 신청 자동 추천: 자원 종류(장비/인력)에 따라 필요한 검사(차량안전/안전교육/건강검진)를 기본 세팅해 원클릭 신청.
- ResourceCheck↔SafetyInspection **연결**: 신청→일정→승인을 한 흐름으로(현재 남남).
- BP용 "현장 자원 준비 현황" 롤업: readiness를 BP 관점(현장별 자원이 투입 대기인지)으로 집계.

**더 편한 화면 구성**
- 네비 **4그룹 재편**: 견적·선정 / 계획서·서명 / 투입·현장 / 서류·검사. (현재 20+ 병렬 메뉴)
- 대시보드를 **관제 화면**으로: 처리대기 큐 + 현장별 파이프라인 단계(서류→검사→투입대기→투입→작업)를 한 화면 보드로.

## 2. 장비공급사

**작업흐름 적합성** — 1·2·3단계(서류 등록·검증·BP 전달)와 7·9단계(실투입 단가확정·거래내역서)의 주역. 크게 개선됨: **후보배지·배차초안 1클릭·OT프리필·투입대기 위젯·거래내역서**. 끊기는 곳: 서류가 **5채널로 분산**(보완/점검/이행지시/수집/심사), 단가가 응찰→배차→투입요청에서 반복 노출.

**더 편리한 자동화**
- 배차+서류묶음 **통합 발송**(현재 2단계 다이얼로그 → 1단계).
- 단가 원장 단일화(응찰→배차→정산 참조) — *단, 단계별 단가차이가 의도인지 결정 필요*.
- 거래내역서 정기 발행(월마감 자동 생성·알림).

**더 편한 화면 구성**
- **서류 허브**(5채널 → 탭 1화면): 받은 요청·상태·업로드를 한 곳에.
- **"내 자원 파이프라인" 보드**: 장비별로 서류→검사→투입대기→투입중→정산 단계를 한 줄에(readiness 확장).

## 3. 인력공급사

**작업흐름 적합성** — 장비공급사와 동일 구조(인력). 개선 동일 적용(readiness·거래내역서·자가관리). **차이**: 인력 배차행에 **OT 단가 컬럼이 없어 정산 OT=0**(월/일대만) — 작업확인서엔 OT시간이 잡히나 금액 반영 안 됨.

**더 편리한 자동화**
- 근무일수 자동파생(이미 적용) + **OT 금액 반영 여부 결정**(인력 OT 단가 도입 시).
- 안전교육·건강검진 만료 임박 인력 자동 리마인드(만료 스케줄러 확장).

**더 편한 화면 구성**
- 장비공급사와 동일: 서류 허브 + 인력 파이프라인 보드. 인력은 **자격/교육 유효기간**을 파이프라인에 함께 표시.

## 4. 협력사 (장비 협력사)

**작업흐름 적합성** — 이번에 **완성도 크게 상승**: 부모 대행 **전체 수정**(A), **자가 로그인+부모 승인**(B), 본인 자원 관리, 본인 거래내역서/정산(D, 본인 격리). 3 온보딩 경로(자가·대행·링크) 모두 실동작. 협력사는 장비공급사와 **같은 화면을 스코프만 좁혀** 사용 → 학습비용 낮음.

**더 편리한 자동화**
- **협력사 온보딩 위저드**: 가입→(부모 승인)→서류 업로드(링크 순서형)→검사 준비까지 단계 안내. 링크 업로드(c경로)와 결합.
- 부모에게 자식 자원 상태 롤업(자식별 readiness·서류 만료·승인 대기)을 한 화면.

**더 편한 화면 구성**
- **SubSuppliersPage를 "협력사 관제"로**: 자식 목록 + 각 자식의 자원·서류·투입대기·가입대기 승인을 한 곳에(가입 승인 UI는 이미 추가됨).
- 협력사 본인 화면: 현행 공급사 화면(스코프) 유지가 정답 — 별도 UI 만들지 말 것(학습비용↑).

## 5. 작업자 (운전자/인력)

**작업흐름 적합성** — 8단계(출근·연장근무→작업확인서) 담당. field-auth 코드 로그인·출근·서명, 단순·명확. 모바일/워치(낙상감지 특허).

**더 편리한 자동화**
- 출근 자동화(현장 지오펜스/워치)로 수기 체크 최소화.
- 작업확인서 **자동 채움**: 출근세션의 시간·OT를 확인서에 프리필(서명만 사람).
- 워치 낙상감지: 실기기 검증 후 Enforce + 주변 전파(특허).

**더 편한 화면 구성**
- 모바일 최소 UI 유지: 출근 버튼 1개 + 확인서 서명. **추가 기능으로 복잡하게 만들지 말 것.**

## 공통 — 가장 큰 레버 (모든 주체 관통)
1. **"자원 파이프라인 상태" 통합 개념**: 자원 1건의 서류→검사→투입대기→투입→작업→정산을 하나의 상태선으로. readiness(C)가 그 첫 조각 — 이걸 각 주체 관점 보드로 확장하면 "지금 어디"가 항상 보임.
2. **네비 재편(BP 4그룹) + 서류 허브(공급사 5채널→탭)**: 이음새·역할전환 대폭 감소.
3. **검사 도메인 연결**(ResourceCheck↔SafetyInspection): 파이프라인 4~5단계의 유일한 실질 갭.

## 자동화 심화 — 결정 필요 (사용자)
- **단가 원장 단일화(B안)**: 단계별 단가차이가 의도(계약가/정산가 분리)인지?
- **인력 OT 단가 도입**: 인력 정산에 OT 금액 반영할지?
- **거래내역서 청구 프로레이션**: 조회기간 비례절단(현재는 계약기간 기준) — 공식 청구서로 쓰려면 필요, 돈 계산 변경이라 결정 필요.
- **수락=배치(C안)**: 배치 진실원천을 작업계획서→수락시점으로 옮길지(최대가치·최대위험).
- **검사 도메인 통합 범위** + 건강검진 날짜통보 신설.

## 자동화 경계 (재확인)
✅ 준비·이동·집계·알림·프리필·초안·롤업 = 자동화. 🔴 전자서명·검사 승인 판정·현장 배치 적합성·반려 사유 = 사람. 자동화는 그 **주변**만.
